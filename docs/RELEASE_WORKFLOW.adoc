= Release Workflow

The repository builds and releases the Tauri application from GitHub Actions.

== Push to main

A push to `main` runs all configured platform builds first.

If every build succeeds, the workflow creates a build tag using the application version from `package.json`:

[source,text]
----
v0.1.0-build.42
----

The build number is the GitHub Actions run number.

The generated release is marked as a prerelease because it came from `main` rather than an explicit version tag.

== Version tags

Pushing a tag matching `v*` creates a normal release:

[source,text]
----
v0.1.1
----

Tag releases are not marked as prereleases.

== Asset publishing

Each platform build uploads its Tauri bundle as an artifact. The publish job downloads those artifacts, packages each platform artifact into a zip, creates or updates the release, and uploads the zips with the workflow attempt number in the filename.

This avoids the fragile release-asset delete/re-upload path that can fail with GitHub API `Not Found` errors.

== pnpm version source of truth

The pnpm version is defined only in `package.json`:

[source,json]
----
"packageManager": "pnpm@9.15.9"
----

The workflow intentionally does not pass a separate pnpm version to `pnpm/action-setup`, avoiding `ERR_PNPM_BAD_PM_VERSION`.
